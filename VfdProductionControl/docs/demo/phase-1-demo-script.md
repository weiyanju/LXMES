# Phase 1 演示脚本

本文档用于验证 VfdProductionControl 第一阶段的模拟生产闭环。演示默认使用内存数据和模拟设备，不需要真实 SQL Server、真实 COM 串口或外部生产系统。

## 准备

1. 在仓库根目录启动应用：

   ```powershell
   dotnet run --project src\VfdControl.App\VfdControl.App.csproj
   ```

2. 确认左侧导航停留在“操作员控制台”。
3. 确认界面提示模拟模式已就绪。

## 操作流程

1. 在员工码输入框输入 `EMP0001`，执行扫描。
2. 在方案列表中选择 `VFD 生产测试演示方案`。
3. 选择 1、2、3 号槽位。
4. 确认槽位选择，进入 VFD 条码绑定。
5. 按提示依次扫描：
   - `VFD202605280001`
   - `VFD202605280002`
   - `VFD202605280003`
6. 确认开始测试。
7. 观察 1、2、3 号槽位并行进入运行状态。
8. 等待模拟步骤完成，检查槽位卡片结论：
   - 绿色表示通过。
   - 红色表示失败。
   - 黄色表示警告。
   - 结论必须同时有文字标签，不能只依赖颜色。
9. 打开左侧“追溯查询”。
10. 使用日期范围、VFD 条码或结论筛选历史运行记录。
11. 打开任意设备运行记录，核对详情：
    - 步骤顺序和步骤名称。
    - 测量数据，例如 VFD 电压和仪表电压。
    - 比对结果和结论。
    - 命令追踪中的请求 JSON 和响应 JSON。

## 通过标准

- 操作员可以从员工码扫描一路完成到测试结论。
- 已选槽位按选择顺序绑定条码。
- 多槽位测试并行执行，单槽位结果不会遮挡其他槽位状态。
- 测试完成后能在追溯查询中找到对应 VFD 条码。
- 设备运行详情能显示步骤、测量、比对和命令追踪。
