# UI Layout Optimization Design

## Goal

将 VfdProductionControl 的 WPF 界面优化为一个带有 iOS HIG 气质的产线控制台：内容优先、层级清晰、状态明确、动作可预期。设计语言参考 Apple Human Interface Guidelines 中的 hierarchy、harmony、consistency、color、typography、materials 和 accessibility 原则，但落地形态必须适合 Windows 产线桌面软件，而不是模拟手机 App。

本次优化关注界面结构、视觉语言、组件规则、中文文案和操作流。领域规则、执行引擎、持久化、模拟设备和真实串口集成不在本设计范围内。

## Scope

覆盖 Phase 1 的 WPF shell 与四个模块：

- 操作员控制台
- 工程维护
- 系统管理
- 追溯查询

文档中的设计要求约束 XAML、ViewModel 暴露给界面的状态字段、公共资源样式和可见中文文案。

## iOS Design Translation

iOS 风格在本项目中不是玻璃拟态、手机底部 Tab、夸张动画或大面积半透明。它被转译为以下产品规则：

1. **层级清楚**：导航、当前任务、主数据、次级详情分层明确。控制元素浮在内容之上，内容本身保持干净。
2. **和谐一致**：圆角、间距、字号、状态色和控件形态形成统一系统，不在不同模块里各自发明样式。
3. **内容优先**：槽位状态、当前步骤、条码、结论和异常信息永远比装饰更重要。
4. **有限色彩**：蓝色表示交互和选中，绿色/黄色/红色只表示生产状态。中性色承担大部分界面面积。
5. **可读优先**：字号、字重、对比度和控件尺寸优先服务产线距离、亮光环境和重复操作。
6. **可感知状态**：所有状态必须使用颜色 + 文本，关键状态再增加图标或形状。
7. **轻量材料感**：使用柔和底色、细边框、轻阴影和圆角表现层级。不要把标准内容区域做成透明玻璃。

## Visual System

### Palette

采用明亮的冷灰中性色，配一个稳定互动蓝：

- `AppBackground`: `#F3F5F8`，主工作区背景。
- `Surface`: `#FBFCFE`，页面和面板表面。
- `SurfaceElevated`: `#FFFFFF`，槽位模块、当前动作、详情面板。
- `SurfaceMuted`: `#E9EDF2`，输入、弱按钮、只读信息带。
- `BorderSubtle`: `#D6DCE4`，常规分割和面板边界。
- `BorderStrong`: `#B8C2CE`，选中、焦点、关键分隔。
- `TextPrimary`: `#1F2933`，主要文本。
- `TextSecondary`: `#4B5563`，说明和次级文本。
- `TextMuted`: `#6B7280`，元信息。
- `InteractionBlue`: `#007AFF`，选中、焦点、主操作。
- `InteractionBluePressed`: `#005BBF`，按下和高强调。
- `PassGreen`: `#34C759`，通过、就绪、完成。
- `WarningAmber`: `#FFB020`，警告、待确认、非阻断异常。
- `FailRed`: `#FF3B30`，失败、停止、危险动作。
- `RunningIndigo`: `#5856D6`，运行中、排队、处理中。

蓝色不得用于非交互装饰文本。状态色不得替代标签。

### Typography

WPF 使用 `Segoe UI, Microsoft YaHei UI, sans-serif`，数据值使用 `Cascadia Mono, Consolas, Microsoft YaHei UI, monospace`。

- 页面标题：24 px, Semibold。
- 模块标题：20 px, Semibold。
- 面板标题：16 px, Semibold。
- 正文和表格：14 px, Regular。
- 状态标签：13 px, Semibold。
- 仪表数据：18 px, Semibold, Mono。

避免超大 hero 字体。生产软件的层级来自清晰分组、字重、留白和状态，而不是营销式标题。

### Shape And Depth

- 常规控件圆角 8 px。
- 小型标签和输入圆角 6 px。
- 槽位模块和详情面板圆角 10 px。
- 标准面板使用 1 px 边框。
- 轻阴影只用于浮层、当前动作面板和选中槽位，不用于所有容器。
- 不使用彩色侧边条作为卡片主识别方式。

### Motion And Feedback

WPF 动效保持克制：

- 按钮 hover、pressed、focus 状态必须明确。
- 运行状态可以使用低频进度或状态点变化。
- 不使用弹跳、夸张位移、长时间循环动画。
- 关键动作完成后给出短文本反馈，例如 `条码已绑定`、`测试已启动`、`停止指令已发送`。

## Shell Layout

Shell 使用两层顶部结构 + 全宽内容区。

```text
全局导航栏: 产品名 | 操作员控制台 | 工程维护 | 系统管理 | 追溯查询 | 模式 | 就绪状态
模块任务栏: 当前模块标题 | 当前任务提示 | 关键状态 | 可选辅助动作
内容区: 当前模块主界面
```

### Global Bar

- 高度 56 到 60 px。
- 左侧显示 `VFD 生产控制`。
- 中部为四个模块入口，使用圆角分段按钮样式。
- 当前模块使用蓝色文字、浅蓝底和清晰边框。
- 右侧显示 `模拟模式`、数据库/设备就绪状态和当前工位提示。
- 导航按钮是切换工作区的控件，不作为装饰标签。

### Module Task Bar

模块任务栏承接 iOS navigation title 的职责，但适合桌面：

- 左侧是当前模块标题。
- 中间是当前任务提示。
- 右侧是与当前模块相关的状态摘要。
- 不在这一行堆放大量表单和命令。

示例：

- 操作员控制台：`扫描员工码，开始生产会话`
- 工程维护：`维护工艺方案、步骤、规则和失败策略`
- 系统管理：`配置工位、槽位、串口、仪表和条码规则`
- 追溯查询：`查询历史运行、步骤、测量、比较和命令记录`

## Module Layout Templates

### Operator Console

操作员控制台采用状态优先布局：

```text
流程进度条
槽位看板                                      当前动作面板
运行明细和指令日志
```

核心规则：

- 槽位看板是视觉中心，永远可见。
- 当前动作面板只突出一个主动作。
- 会话、方案、条码和启动流程按照状态逐步展开。
- 运行中禁用会造成流程混乱或安全风险的设置动作。
- 完成后显示结论摘要和追溯入口。

操作步骤：

1. 等待员工码：显示员工码输入和扫描/登录动作。
2. 选择方案：显示可执行方案，单一方案可自动选中。
3. 选择槽位：显示槽位看板和确认动作。
4. 绑定条码：显示当前槽位提示、条码输入和绑定动作。
5. 确认启动：显示选中槽位、方案版本和启动动作。
6. 运行中：显示槽位状态、当前步骤、测量和异常。
7. 已完成：显示总通过率、失败槽位、警告槽位和追溯入口。

槽位模块必须显示：

- 槽位号、COM 口、选择状态。
- VFD 条码和绑定状态。
- 当前状态：待选择、待扫码、已绑定、排队、运行中、通过、失败、警告、停止。
- 当前步骤和进度。
- 最新测量、比较结果或最新命令。
- 最终结论。

### Engineering

工程维护使用三栏布局：

```text
方案与版本列表 | 步骤表和流程编辑 | 当前步骤参数与失败策略
```

设计规则：

- 左侧列表保持紧凑，适合快速切换方案。
- 中央步骤表是主内容，支持清晰的顺序、步骤类型、设备来源、判定规则和状态。
- 右侧面板只展示当前选中步骤的可编辑属性。
- 保存、发布、停用、版本回滚等动作必须带状态反馈。
- 工程复杂度留在工程区，不泄露到操作员日常流程。

### Administration

系统管理使用配置对象布局：

```text
配置对象列表 | 当前配置表 | 详情与校验
```

对象包括工位、槽位、串口、仪表、点位和条码规则。编辑控件应靠近被编辑对象，避免把所有设置堆到顶部工具栏。

管理界面可以比操作员界面更密集，但必须保留：

- 当前选中对象。
- 未保存状态。
- 校验错误。
- 保存成功或失败反馈。
- 可追踪的配置变更入口。

### Traceability

追溯查询使用查询 + 详情布局：

```text
筛选和运行列表 | 运行详情
```

左侧包含筛选条件和结果列表。右侧展示选中运行的步骤、测量、比较、规则快照和命令请求/响应 JSON。

详情区可以滚动和密集，但不要使用多层嵌套卡片。优先使用分组标题、表格、折叠段和只读数据块。

## Component Rules

### Buttons

- 同一区域只能有一个主按钮。
- 主按钮使用互动蓝，表示下一步主动作。
- 次按钮使用浅色表面和边框。
- 停止、删除、危险动作使用红色或警告样式，并保留明确文案。
- 常用动作按钮高度不低于 40 px，生产关键动作不低于 44 px。

### Inputs

- 输入框使用浅色背景、1 px 边框和明确焦点态。
- 标签使用短中文。
- 员工码和条码输入支持键盘快速录入。
- 校验失败显示错误文本，不只改变边框颜色。

### Tables

- 表头固定、中文清晰。
- 状态、时间、动作列使用稳定宽度。
- 技术值保留英文和符号，例如 VFD、COM、JSON、Modbus、RTU。
- 表格行高保持紧凑，但不能压缩到影响点击和阅读。

### Status

状态必须使用文本 + 色彩，关键状态增加图标或形状：

- `通过`: 绿色。
- `失败`: 红色。
- `警告`: 黄色。
- `运行中`: 靛蓝或蓝色。
- `待扫码`、`未绑定`、`待选择`: 中性色。
- `停止`: 红色或深中性色，按上下文区分人工停止和异常停止。

### Dialogs And Secondary Workspaces

工程维护、系统管理和追溯查询可以作为主模块切换，也可以在操作员控制台上方以独立窗口打开。无论哪种方式，运行中的操作员控制台状态必须可恢复，不能因为进入配置页面丢失当前会话。

确认类弹窗只用于危险动作、不可逆保存、停止运行和异常收尾。常规编辑优先使用内联反馈。

## Chinese Copy Rules

- 可见 UI 文案使用简体中文。
- 技术名词保留英文：VFD、COM、SQL Server、JSON、Modbus、RTU。
- 按钮文案使用动词 + 对象，例如 `绑定条码`、`启动测试`、`保存版本`。
- 状态文案使用短词，例如 `待扫码`、`运行中`、`通过`、`失败`。
- 如发现终端或构建链路显示乱码，先确认文件实际编码。Markdown 和 XAML 文件应保持 UTF-8。

## Implementation Strategy

1. 抽取公共颜色、字号、圆角、边框、按钮、输入、表格和状态样式。
2. 将 shell 调整为顶部全局导航栏 + 模块任务栏。
3. 将操作员控制台改为流程进度、槽位看板、当前动作面板和运行明细结构。
4. 将工程维护、系统管理、追溯查询分别对齐三栏、配置对象、查询详情模板。
5. 清理可见中文文案，保留技术名词。
6. 用构建、测试和 WPF 启动验证界面能运行。

## Acceptance Criteria

- 主导航位于顶部，当前模块状态清楚。
- 操作员控制台只突出一个下一步主动作。
- 槽位状态、条码、步骤、测量和结论在生产过程中可见。
- 四个模块使用一致的颜色、字号、圆角、按钮、输入、表格和状态语言。
- 状态不只依赖颜色。
- 中文文案在 UI 中可读，文件保持 UTF-8。
- `dotnet build VfdProductionControl.sln` 通过。
- `dotnet test VfdProductionControl.sln` 通过。
- `dotnet run --project src\VfdControl.App\VfdControl.App.csproj` 能启动应用。
